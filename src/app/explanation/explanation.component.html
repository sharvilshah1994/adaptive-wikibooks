Technology Used

  • Elastic Search: Lucene-based Indexing
  • Mongo DB: Storage
  • Python: Scraping
  • Angular 4: Web Application
  • Amazon Web Services (AWS): Host Elastic Search and Web Application

Steps taken to implement this:

The following steps were taken to implement this

  1. Crawled java programming wikibooks pages and Oracle JavaTM tutorial Pages. URL: https://en.wikibooks.org/wiki/Java_Programming
    URL: https://docs.oracle.com/javase/tutorial/
    • This task was done using Python library Scrapy. Scrapy crawled through the given link and scraped all the relevant data.

  2. The scraped data is put to Mongo DB so that Encoding and Decoding errors do not take place.
    • Pymongo library is used to dump the scraped data to Mongo.

  3. From Mongo DB, I have used Elastic Search library of python to create schema, index data and store all the data in Elastic Search Schema. Indexing is NOT done page wise. Indexing is done sub-heading wise to get user a better idea. The data between two h2 HTML tags is stored with Topic name, Sub-heading, Data and Link.
    • Schema for Elastic Search is something like this:
      • Topic: (Datatype=TEXT); Stores the chapter name of the book
      • Link: (Datatype=TEXT); Stores the Wiki link to the book
      • Heading: (Datatype=TEXT); Stores the sub-heading of the book page
      • Content: (Datatype=TEXT); Content under a specific sub-heading

  4. Angular 4: JavaScript framework. This is used to implement the web application.

  5. Hosted Elastic Search and Web application on Amazon Web Services (AWS).
